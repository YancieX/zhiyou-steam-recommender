# 需求规格说明书

## 一、引言

### 1.1 编写目的

本文档为"智荐优游"Steam游戏推荐系统的需求规格说明书，旨在明确系统功能需求、性能需求及非功能性需求，作为后续开发、测试和项目验收的依据。

### 1.2 项目范围

本项目为纯前端Steam游戏推荐网站，主要功能范围包括：

- 用户注册与登录
- 游戏浏览与搜索
- 个性化游戏推荐
- 愿望单管理与降价提醒
- Steam账号游戏库导入
- 管理后台

本项目不包括：真实支付功能、后端服务器、多人社交功能。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| 协同过滤 | 推荐算法的一种，通过分析用户行为相似度进行推荐 |
| API | 应用程序接口，是一种允许不同软件系统之间进行通信和交互的工具 |
| localStorage | 浏览器本地存储机制 |
| HMR | 热模块替换，Development模式下代码修改时自动更新 |

### 3.3 用例描述

#### 3.3.1 用户注册

| 项目 | 内容 |
|------|------|
| 用例ID | uid001 |
| 用例名称 | 用户注册 |
| 参与者 | 游客 |
| 前置条件 | 游客已进入注册页面 |
| 后置条件 | 注册成功，系统自动登录并跳转至首页 |
| 优先级 | Must Have |

**基本流程：**
1. 游客输入用户名、邮箱、密码
2. 系统验证邮箱是否已被注册
3. 邮箱未注册，创建用户账号
4. 系统自动登录，跳转至首页

**异常流程：**
- 邮箱已存在 → 提示"邮箱已被注册"
- 密码长度不足6位 → 提示"密码至少6位"
- 邮箱格式不符规范 → 提示"邮箱格式错误"

## 四、非功能性需求

### 4.1 性能需求

| 指标 | 目标 | 备注 |
|------|------|------|
| 首屏加载 | < 3s | 正常网络环境下 |
| 页面切换 | < 500ms | 本地路由跳转 |
| 推荐计算 | < 6s | 基于50款游戏 |
| 搜索响应 | < 10s | 包含API请求时间 |

### 4.2 安全性需求

- 用户密码采用哈希存储，不明文保存
- 用户数据仅存储于本地，不上传至服务器
- Steam API Key仅在本地使用，不发送至第三方
- 用户输入内容进行基本XSS防护

### 4.3 可用性需求

- 界面布局清晰直观，降低用户学习成本
- 操作结果提供明确反馈
- 加载过程显示加载动画，避免用户误解
- 错误场景提供友好的提示信息

### 4.4 兼容性需求

- windows可用
- 支持 Edge 及一般浏览器
- 适配 1920x1080 桌面分辨率
- 移动端基本可用

### 4.5 可维护性需求

- 代码添加必要注释，关键函数说明功能
- 采用组件化开发原则
- 命名规范统一，语义清晰
- 目录结构清晰，便于定位文件

## 五、约束与假设

### 5.1 约束条件

1. **纯前端架构**：无后端服务器，所有数据存储于浏览器
2. **存储限制**：localStorage容量约5MB，数据存储受限
3. **跨域限制**：Steam API存在跨域限制，需通过代理解决
4. **API限流**：Steam API存在调用频率限制
5. **开发周期**：课程设计周期，约1周

### 5.2 假设条件

1. 用户使用Edge等现代浏览器
2. 用户具备基本计算机操作能力
3. Steam API服务正常运行
4. 网络连接基本稳定
5. 游戏数据版权归Steam和开发商所有
