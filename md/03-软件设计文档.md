# 软件设计文档

## 一、总体设计

### 1.1 系统架构

本项目采用纯前端SPA（Single Page Application）架构，前端应用运行于用户浏览器，无需后端服务器支持。

```
┌─────────────────────────────────────────────────┐
│                  用户浏览器                        │
│  ┌───────────────────────────────────────────┐  │
│  │             React 应用 (SPA)              │  │
│  ├───────────────────────────────────────────┤  │
│  │  页面层 (Pages)                            │  │
│  │  首页 | 详情 | 搜索 | 愿望单 | 个人中心     │  │
│  ├───────────────────────────────────────────┤  │
│  │  组件层 (Components)                       │  │
│  │  导航栏 | 游戏卡片 | 弹窗 | Toast          │  │
│  ├───────────────────────────────────────────┤  │
│  │  状态层 (Stores - Zustand)                 │  │
│  │  用户 | 游戏 | 愿望单 | 设置               │  │
│  ├───────────────────────────────────────────┤  │
│  │  工具层 (Utils)                            │  │
│  │  推荐算法 | Steam API | 存储               │  │
│  └───────────────────────────────────────────┘  │
│                      │                          │
│  ┌─────────────────┴───────────────────────┐   │
│  │           localStorage (本地存储)        │   │
│  │  用户 | 游戏 | 评分 | 愿望单 | 缓存       │   │
│  └─────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
          │                │
          │                ▼
          │       ┌─────────────────┐
          └──────▶│   Steam API     │
                  └─────────────────┘
```

### 1.2 技术选型

| 类别 | 技术 | 版本 | 选择理由 |
|------|------|------|----------|
| 前端框架 | React | 18.x | 生态成熟，组件化开发 |
| 开发语言 | TypeScript | 5.x | 类型安全，降低错误率 |
| 构建工具 | Vite | 6.x | 启动快速，热更新体验好 |
| 样式方案 | TailwindCSS | 3.x | 原子化CSS，提升开发效率 |
| 状态管理 | Zustand | 5.x | 轻量级，API简洁 |
| 路由管理 | React Router | 7.x | 官方推荐路由方案 |
| 图标库 | Lucide React | 最新 | 图标丰富，样式统一 |

### 1.3 第三方服务

| 服务 | 用途 |
|------|------|
| Steam Store API | 游戏搜索、详情、价格查询 |
| Steam Web API | 用户游戏库导入（需API Key） |
| corsproxy.io | 跨域代理服务 |
| images.weserv.nl | 图片代理服务 |

### 1.4 存储方案

本项目采用浏览器localStorage进行数据持久化，纯前端实现，无需后端支持。

- 优点：实现简单，无需服务器
- 缺点：容量有限（约5MB），不支持多设备同步

### 2.4 协同过滤算法伪代码

```
函数 协同过滤推荐(目标用户ID, 所有游戏, 所有评分, 所有用户):
    // 1. 构建用户-游戏评分矩阵
    为每个用户创建评分向量

    // 2. 计算目标用户与其他用户的相似度
    对每个其他用户:
        计算余弦相似度(目标用户向量, 其他用户向量)

    // 3. 找出最相似的K个用户
    按相似度排序，取前K个

    // 4. 计算候选游戏的预测评分
    对每个目标用户未评分的游戏:
        加权平均分 = Σ(相似用户评分 × 相似度) / Σ相似度

    // 5. 返回评分最高的N个游戏
    按预测评分排序，返回前N个
```

## 三、接口设计

### 3.1 内部接口（Store层）

#### 3.1.1 UserStore

```typescript
// 登录
login(email: string, password: string): boolean

// 注册
register(username: string, email: string, password: string): boolean

// 退出登录
logout(): void

// 修改资料
updateProfile(updates: Partial<User>): void
```

#### 3.1.2 GameStore

```typescript
// 购买游戏
purchaseGame(userId: string, gameId: string, game?: Game): void

// 评分
rateGame(userId: string, gameId: string, score: number, playtimeHours: number): void

// 获取用户购买的游戏
getUserPurchasedGames(userId: string): Game[]

// 同步Steam价格
syncPricesWithSteam(): Promise<{success: number, failed: number, total: number}>
```

#### 3.1.3 WishlistStore

```typescript
// 添加愿望单
addToWishlist(userId: string, gameId: string, game?: Game): void

// 移除愿望单
removeFromWishlist(userId: string, gameId: string): void

// 是否已在愿望单
isInWishlist(userId: string, gameId: string): boolean

// 检查降价
checkPriceChanges(): PriceAlert[]
```

### 3.2 外部API接口

#### 3.2.1 Steam Store 搜索接口

```
GET https://store.steampowered.com/api/storesearch
参数:
  - term: 搜索关键词
  - l: 语言 (schinese)
  - cc: 国家 (cn)
  - count: 返回数量
返回: 游戏搜索结果列表
```

#### 3.2.2 Steam 游戏详情接口

```
GET https://store.steampowered.com/api/appdetails
参数:
  - appids: 游戏ID（可多个）
  - l: 语言
  - cc: 国家
返回: 游戏详细信息
```

#### 3.2.3 Steam 用户游戏库接口

```
GET https://api.steampowered.com/IPlayerService/GetOwnedGames/v1/
参数:
  - key: Steam API Key
  - steamid: 用户Steam ID
  - include_appinfo: 是否包含游戏信息
返回: 用户拥有的游戏列表和游玩时长
```
