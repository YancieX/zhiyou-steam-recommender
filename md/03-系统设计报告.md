# 智荐优游 - 系统设计报告

## 1. 系统架构设计

### 1.1 总体架构

本系统采用纯前端单页应用（SPA）架构，所有数据存储在浏览器本地，不依赖后端服务器。系统基于 React 18 + TypeScript + Vite 构建，使用 Zustand 进行状态管理，React Router 处理路由，TailwindCSS 实现样式。

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户浏览器                                │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                    React 前端应用 (SPA)                    │  │
│  ├───────────────────────────────────────────────────────────┤  │
│  │  页面层 (Pages)                                            │  │
│  │  首页 | 游戏详情 | 搜索 | 愿望单 | 个人中心 | 管理后台     │  │
│  ├───────────────────────────────────────────────────────────┤  │
│  │  组件层 (Components)                                       │  │
│  │  导航栏 | 游戏卡片 | 评分弹窗 | 价格提醒 | Toast          │  │
│  ├───────────────────────────────────────────────────────────┤  │
│  │  状态层 (Stores - Zustand)                                 │  │
│  │  用户状态 | 游戏状态 | 愿望单状态 | 设置状态 | Toast状态   │  │
│  ├───────────────────────────────────────────────────────────┤  │
│  │  工具层 (Utils)                                            │  │
│  │  推荐算法 | Steam API | 存储工具 | AI推荐                 │  │
│  ├───────────────────────────────────────────────────────────┤  │
│  │  数据层 (Data)                                             │  │
│  │  类型定义 | 模拟数据                                       │  │
│  └───────────────────────────────────────────────────────────┘  │
│                           │                                    │
│  ┌────────────────────────┴──────────────────────────────┐    │
│  │                  localStorage                          │    │
│  │  用户数据 | 游戏数据 | 评分数据 | 愿望单数据 | 缓存     │    │
│  └───────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
                           │
                           │ HTTP 请求
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│                     外部 API 服务                                │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐      │
│  │  Steam Store │  │ Steam Web API│  │  图片代理服务     │      │
│  │     API      │  │              │  │                  │      │
│  └──────────────┘  └──────────────┘  └──────────────────┘      │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 架构特点

1. **前后端分离**：纯前端架构，数据本地存储
2. **组件化设计**：React 组件化开发，高内聚低耦合
3. **状态集中管理**：Zustand 统一管理应用状态
4. **模块化工具**：工具函数按功能模块划分
5. **类型安全**：TypeScript 全程类型检查

### 2.2 关键库说明

#### 2.2.1 Zustand 状态管理
- 轻量级：相比 Redux 更简洁，更少的样板代码
- 高性能：精确的组件更新，避免不必要的重渲染
- 简单易用：API 直观，学习成本低
- TypeScript 友好：完美的类型支持

#### 2.2.2 TailwindCSS
- 原子化 CSS：减少 CSS 文件体积
- 开发效率：无需切换文件，直接在 HTML 中写样式
- 响应式：内置断点，快速实现响应式布局
- 主题定制：通过配置文件自定义主题

#### 2.2.3 Vite
- 极速启动：基于 ESM 的开发服务器，秒级启动
- 热更新：HMR 速度快，开发体验好
- 优化构建：Rollup 构建，自动代码分割
- 插件丰富：生态完善，扩展性强

### 5.2 外部 API 接口

#### 5.2.1 Steam Store Search API

- **URL**: `https://store.steampowered.com/api/storesearch`
- **方法**: GET
- **参数**: 
  - `term`: 搜索关键词
  - `l`: 语言（english/schinese）
  - `cc`: 国家代码（cn/us）
  - `count`: 返回数量
- **返回**: 搜索结果列表（游戏 ID、名称、价格、缩略图等）

#### 5.2.2 Steam Store App Details API

- **URL**: `https://store.steampowered.com/api/appdetails`
- **方法**: GET
- **参数**:
  - `appids`: 游戏 App ID（支持多个，逗号分隔）
  - `l`: 语言
  - `cc`: 国家代码
- **返回**: 游戏详细信息（描述、评分、截图、价格等）

#### 5.2.3 Steam Web API - GetOwnedGames

- **URL**: `https://api.steampowered.com/IPlayerService/GetOwnedGames/v1/`
- **方法**: GET
- **参数**:
  - `key`: Steam API Key
  - `steamid`: Steam 用户 ID
  - `include_appinfo`: 是否包含游戏信息
  - `include_played_free_games`: 是否包含免费游戏
- **返回**: 用户拥有的游戏列表及游玩时长

## 6. 推荐系统设计

### 6.1 推荐算法架构

```
用户数据（已购买 + 评分 + 愿望单）
        │
        ▼
┌───────────────────┐
│  用户偏好分析      │
│  - 类型偏好       │
│  - 标签偏好       │
│  - 内容关键词     │
└───────────────────┘
        │
        ├──────────────────────┐
        ▼                      ▼
┌──────────────┐       ┌──────────────┐
│  AI 内容推荐  │       │ 协同过滤推荐 │
│  - 类型匹配   │       │  - 用户相似度 │
│  - 标签匹配   │       │  - 余弦相似度 │
│  - 内容相似度 │       │  - 游玩时长权重 │
│  - 热度 + 评分│       │              │
└──────────────┘       └──────────────┘
        │                      │
        └──────────┬───────────┘
                   ▼
          ┌──────────────┐
          │  混合排序策略  │
          │  - 匹配度优先 │
          │  - 热度次之   │
          │  - 评分再次之 │
          │  - 评分高可弥补热度 │
          └──────────────┘
                   │
                   ▼
          推荐结果列表（去重 + 排除已购买）
```

### 6.2 推荐算法流程

1. **数据收集**：收集用户已购买游戏、评分记录、愿望单游戏
2. **偏好分析**：分析用户的类型偏好、标签偏好、内容关键词
3. **候选池构建**：从 Steam 搜索 API 获取全范围游戏候选
4. **评分计算**：
   - AI 推荐：计算匹配度分数（类型+标签+内容相似度）
   - 协同过滤：计算用户相似度，推荐相似用户喜欢的游戏
5. **综合排序**：按匹配度 > 热门程度 > 评分的优先级排序
6. **过滤去重**：排除已购买游戏、已推荐过的游戏
7. **返回结果**：返回 Top N 推荐结果

### 6.3 评分权重分配

| 维度 | AI 推荐权重 | 说明 |
|------|------------|------|
| 类型匹配 | 30 分 | 游戏类型是否匹配用户偏好 |
| 标签匹配 | 每标签 6 分 | 游戏标签与用户偏好标签的匹配 |
| 内容相似度 | 最高 30 分 | 游戏描述文本与偏好的内容相似度 |
| 促销加成 | 8 分 | 促销中且折扣 ≥ 30% |
| 热门程度 | 0-20 分 | Steam 好评率反映的热度 |
| 评分质量 | 0-15 分 | 游戏评分高低 |
| 质量加成 | 0-8 分 | 高评分佳作的额外加成 |

## 8. 安全设计

### 8.1 数据安全

- **密码存储**：使用哈希函数存储密码，不明文保存
- **本地存储**：数据仅保存在用户浏览器，不上传服务器
- **API Key**：Steam API Key 仅本地使用，不发送给第三方
- **输入验证**：所有用户输入进行前端校验和转义

### 8.2 XSS 防护

- 使用 React 自动转义
- 对 dangerouslySetInnerHTML 内容进行净化
- 对用户生成内容进行 HTML 转义

### 8.3 依赖安全

- 定期更新依赖包
- 使用 npm audit 检查漏洞
- 只使用可信来源的第三方库

## 9. 性能设计

### 9.1 性能优化策略

1. **代码分割**：按路由分割代码，按需加载
2. **图片优化**：使用图片代理压缩，懒加载
3. **缓存策略**：游戏详情缓存，避免重复请求
4. **虚拟列表**：长列表使用虚拟滚动（可选优化）
5. **防抖节流**：搜索输入防抖，滚动监听节流
6. **状态优化**：Zustand 精确更新，避免不必要重渲染

### 9.2 性能指标

| 指标 | 目标值 |
|------|--------|
| 首屏加载 (LCP) | < 2.5s |
| 首次输入延迟 (FID) | < 100ms |
| 累计布局偏移 (CLS) | < 0.1 |
| JS 包大小 (gzip) | < 200KB |
| 推荐计算时间 | < 500ms |
