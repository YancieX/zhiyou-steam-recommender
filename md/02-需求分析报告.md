# 智荐优游 - 需求分析报告

## 1. 项目概述

### 1.1 项目背景

Steam 作为全球最大的 PC 游戏平台，拥有超过 10 万款游戏和 1.2 亿月活跃用户。随着游戏数量的持续增长，用户面临着"信息过载"的问题——如何在海量游戏中找到符合自己口味的作品。同时，Steam 平台频繁的促销活动虽然为玩家提供了实惠，但也让用户难以在短时间内发现值得购买的优惠游戏。

### 1.2 项目目标

"智荐优游"是一款基于协同过滤算法的 Steam 游戏促销个性化推荐与愿望单降价提醒系统。系统通过分析用户的游戏购买行为、评分和游玩时长，结合混合推荐算法（协同过滤 + AI 内容分析），为用户精准推荐符合其偏好的游戏，同时提供愿望单价格监控和降价提醒功能。

### 1.3 目标用户

| 用户角色 | 描述 | 核心需求 |
|----------|------|----------|
| 普通玩家 | Steam 游戏爱好者，喜欢购买和体验新游戏 | 发现符合口味的好游戏，不错过促销优惠 |
| 价格敏感型用户 | 关注游戏性价比，习惯等降价再买 | 愿望单降价提醒，促销游戏推荐 |
| 游戏收藏者 | 喜欢收藏游戏，有大量愿望单 | 愿望单管理，批量监控价格 |
| 管理员 | 系统运营维护人员 | 价格管理、数据统计、系统配置 |

### 1.4 项目范围

**包含范围：**
- 用户注册登录与个人资料管理
- Steam 账号数据导入
- 游戏浏览与搜索
- 个性化游戏推荐（AI 推荐 + 协同过滤）
- 愿望单管理与降价提醒
- 游戏评分与游玩时长记录
- 模拟购买功能
- 管理后台（价格管理、数据统计）

**不包含范围：**
- 真实支付与交易功能
- 后端服务器（纯前端演示）
- 多设备数据同步
- 社区社交功能

## 2. 功能需求

### 2.1 用户模块

#### 2.1.1 用户注册
- 用户通过邮箱、用户名、密码注册账号
- 邮箱格式校验
- 密码长度校验（至少 6 位）
- 用户名唯一性校验
- 注册成功自动登录

#### 2.1.2 用户登录
- 邮箱 + 密码登录
- 登录状态保持（刷新页面不丢失）
- 记住登录状态（7 天内自动登录）
- 登录失败提示

#### 2.1.3 用户退出
- 一键退出登录
- 退出后返回首页

#### 2.1.4 个人资料管理
- 查看和修改用户名
- 查看和修改头像
- 修改个人简介
- 修改密码
- 绑定 Steam 账号

### 2.2 Steam 账号导入模块

#### 2.2.1 Steam API Key 配置
- 提供 Steam API Key 申请引导
- 支持用户输入并保存 API Key
- API Key 本地存储，不上传服务器

#### 2.2.2 游戏库导入
- 导入用户 Steam 游戏库（已拥有的游戏）
- 导入游戏游玩时长
- 导入游戏最后游玩时间
- 导入进度显示
- 导入失败错误提示

#### 2.2.3 Steam 资料同步
- 同步 Steam 昵称
- 同步 Steam 头像
- 同步 Steam 个人资料

### 2.3 游戏浏览模块

#### 2.3.1 游戏列表展示
- 游戏卡片展示（封面、名称、价格、折扣、评分）
- 网格布局，响应式排列
- 支持按不同维度浏览
- 分页/滚动加载

#### 2.3.2 游戏搜索
- 关键词搜索游戏
- 实时搜索建议
- 搜索结果分页加载（触底加载更多）
- 搜索历史记录
- 搜索结果显示价格和折扣

#### 2.3.3 游戏详情
- 游戏封面和背景图
- 游戏名称、开发商、发行商
- 游戏描述（关于这款游戏）
- 游戏标签和类型
- 发售日期
- 价格信息（原价、促销价、折扣）
- Steam 评分和评价数
- 游戏截图
- 支持平台（Windows/Mac/Linux）
- Metacritic 评分

### 2.4 推荐系统模块

#### 2.4.1 AI 推荐
- 基于用户已购买游戏进行偏好分析
- 基于用户评分进行偏好分析
- 基于愿望单游戏进行偏好分析
- 基于游戏类型匹配推荐
- 基于游戏标签匹配推荐
- 基于游戏内容相似度推荐（关于这款游戏文本分析）
- 考虑热门程度和评分的权衡
- 每次推荐不重复之前推荐过的游戏
- 从全 Steam 范围推荐，不限于精选游戏

#### 2.4.2 协同过滤推荐
- 基于用户的协同过滤算法
- 余弦相似度计算用户相似度
- 推荐相似用户喜欢的游戏
- 结合游玩时长权重
- 发现用户可能感兴趣的冷门佳作

#### 2.4.3 推荐模式切换
- 支持 AI 推荐和协同过滤两种模式切换
- 展示两种推荐模式的对比说明
- 用户可自由切换查看不同推荐结果

#### 2.4.4 推荐说明
- 显示推荐匹配度百分比
- 显示推荐理由（为什么推荐这款游戏）
- 显示匹配维度（类型、标签、内容主题等）
- 显示 AI 分析文本

### 2.5 愿望单模块

#### 2.5.1 愿望单管理
- 添加游戏到愿望单
- 从愿望单移除游戏
- 愿望单列表展示
- 愿望单游戏数量显示
- 心形图标标识愿望单状态

#### 2.5.2 价格监控
- 记录愿望单添加时的价格
- 监控价格变化
- 降价时标记提醒状态

#### 2.5.3 降价提醒
- 降价游戏醒目展示
- 显示原价和现价对比
- 显示降价幅度
- 降价提醒动画效果
- 一键清除已读提醒

### 2.6 评分与购买模块

#### 2.6.1 模拟购买
- 一键模拟购买游戏
- 购买后游戏进入游戏库
- 可设置购买来源（手动/Steam 导入）

#### 2.6.2 游戏评分
- 对已购买游戏进行评分（1-5 星）
- 记录游玩时长
- 提交评分时间
- 支持修改评分

#### 2.6.3 游戏库管理
- 查看已购买游戏列表
- 按购买时间排序
- 显示游玩时长
- 显示评分

### 2.7 管理后台模块

#### 2.7.1 价格管理
- 手动调整游戏价格
- 随机调整价格（模拟促销）
- 价格同步到 Steam 实时价格
- 价格同步进度显示

#### 2.7.2 数据统计
- 游戏总数统计
- 用户总数统计
- 评分总数统计
- 愿望单总数统计
- 最后价格同步时间

#### 2.7.3 数据管理
- 清空统计数据功能
- 二次确认防止误操作
- 清空后自动刷新页面

### 2.8 系统设置模块

#### 2.8.1 主题设置
- 深色/浅色主题切换
- 主题偏好保存

#### 2.8.2 通知设置
- 降价提醒开关
- 推荐通知开关

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首屏加载时间 | < 2s | 网络正常情况下 |
| 页面切换时间 | < 300ms | 页面间导航 |
| 推荐计算时间 | < 1s | 基于 50 款游戏的推荐计算 |
| 搜索响应时间 | < 500ms | 关键词搜索响应 |
| 图片加载时间 | < 1s | 单张游戏封面图 |
| 并发用户数 | 不适用 | 纯前端系统，无服务端压力 |

### 3.2 可用性需求

- **响应式设计**：支持桌面端（≥1024px）、平板（768px-1023px）、手机（<768px）
- **浏览器兼容**：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- **离线可用**：基本功能可在离线状态下使用（已缓存数据）
- **错误处理**：友好的错误提示，不暴露技术细节
- **加载状态**：所有异步操作都有加载动画提示

### 3.3 安全性需求

- **数据存储**：用户数据存储在浏览器 localStorage，不上传服务器
- **密码安全**：密码使用哈希存储，不明文保存
- **输入验证**：所有用户输入进行前端校验
- **XSS 防护**：对用户输入内容进行转义处理
- **API Key 保护**：Steam API Key 仅保存在本地，不发送到第三方

### 3.4 可维护性需求

- **代码规范**：遵循 ESLint 代码规范
- **类型安全**：使用 TypeScript 确保类型安全
- **模块化设计**：组件化、模块化，便于扩展
- **注释完整**：关键逻辑有清晰注释
- **文档齐全**：有完整的技术文档和使用说明

### 3.5 用户体验需求

- **界面美观**：深色赛博朋克风格，符合游戏用户审美
- **操作流畅**：动画过渡自然，无卡顿感
- **学习成本低**：界面直观，新用户快速上手
- **反馈及时**：用户操作有即时视觉反馈
- **减少等待**：加载状态有进度提示或骨架屏

## 6. 约束与假设

### 6.1 约束条件

1. **纯前端架构**：本系统为纯前端演示系统，不包含后端服务器
2. **数据存储**：所有数据存储在浏览器 localStorage，容量有限（约 5MB）
3. **API 限制**：Steam API 有调用频率限制，需注意控制请求量
4. **跨域限制**：浏览器同源策略限制，需通过代理解决
5. **离线限制**：无网络时无法获取最新游戏数据和价格

### 6.2 假设条件

1. 用户使用现代浏览器（支持 ES6+、localStorage）
2. 用户有基本的计算机操作能力
3. Steam API 服务稳定可用
4. 游戏数据版权归 Steam 和游戏开发商所有
5. 本系统仅用于演示和学习目的
