# 智荐优游 - 详细设计说明书

## 1. 引言

### 1.1 编写目的

本文档为智荐优游的详细设计说明书，旨在对系统各模块的详细实现进行说明，为开发人员提供编码依据。文档涵盖了系统的整体结构、模块设计、算法实现、数据结构等内容。

### 1.2 项目背景

"智荐优游"是一款基于协同过滤算法的Steam游戏促销个性化推荐与愿望单降价提醒系统。系统采用纯前端架构，使用React+TypeScript+Vite构建，所有数据存储在浏览器localStorage中。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| SPA | 单页应用 |
| CF | 协同过滤 |
| User-based CF | 基于用户的协同过滤 |
| 余弦相似度 | 向量空间中两个向量夹角的余弦值 |
| Zustand | 轻量级 React 状态管理库 |
| HMR | 热模块替换 |

## 3. 用户模块详细设计

### 3.1 用户状态管理

#### 3.1.1 状态定义

```typescript
interface UserState {
  currentUser: User | null;      // 当前登录用户
  isAuthenticated: boolean;       // 是否已认证
  isAdmin: boolean;               // 是否管理员
  users: User[];                  // 用户列表
}
```

#### 3.1.2 核心方法

**initialize()**
- 功能：初始化用户状态，从 localStorage 加载数据
- 流程：
  1. 从 storage 读取用户列表
  2. 从 storage 读取当前登录用户
  3. 设置 isAuthenticated 和 isAdmin 标志

**login(email, password)**
- 功能：用户登录
- 参数：
  - email: 邮箱
  - password: 密码（明文，与哈希比较）
- 返回：boolean - 是否登录成功
- 流程：
  1. 在用户列表中查找匹配 email 和 passwordHash 的用户
  2. 找到则保存到 currentUser，设置认证状态
  3. 未找到返回 false

**register(username, email, password, adminCode?)**
- 功能：用户注册
- 参数：
  - username: 用户名
  - email: 邮箱
  - password: 密码
  - adminCode: 管理员注册码（可选）
- 返回：boolean - 是否注册成功
- 流程：
  1. 检查邮箱是否已存在
  2. 生成唯一用户 ID
  3. 密码直接作为 passwordHash（演示系统，简化处理）
  4. 判断是否管理员注册码，设置 role
  5. 保存用户并自动登录

### 3.2 密码哈希说明

> **注意**：本系统为纯前端演示系统，密码使用简化存储方式。
> 生产环境应使用 bcrypt 等安全哈希算法，并在后端进行验证。

#### 4.1.2 核心方法

**purchaseGame(userId, gameId, game?, source?)**
- 功能：模拟购买游戏
- 参数：
  - userId: 用户 ID
  - gameId: 游戏 ID
  - game: 游戏对象（可选，游戏不在库中时自动添加）
  - source: 来源（manual / steam_import）
- 流程：
  1. 检查是否已购买
  2. 如果游戏不在库中且提供了 game 对象，添加到游戏库
  3. 创建购买记录
  4. 保存到 localStorage

- 功能：添加浏览记录

  - addBrowsedGame：临时浏览记录，不影响游戏库

**rateGame(userId, gameId, score, playtimeHours)**
- 功能：对游戏评分
- 参数：
  - score: 评分（1-5）
  - playtimeHours: 游玩时长（小时）
- 流程：
  1. 检查是否已购买
  2. 查找已有评分记录
  3. 更新或创建评分记录
  4. 保存到 localStorage

**syncPricesWithSteam()**
- 功能：从 Steam 同步最新价格
- 返回：{ success, failed, total }
- 流程：
  1. 收集需要同步的游戏 ID
  2. 批量调用 Steam Store API
  3. 更新游戏价格信息
  4. 触发降价提醒检查

## 6. 推荐系统详细设计

### 6.1 协同过滤推荐算法

#### 6.1.1 算法原理

基于用户的协同过滤（User-based Collaborative Filtering）：
1. 找到与目标用户品味相似的用户群体
2. 推荐这些相似用户喜欢但目标用户还没玩过的游戏

#### 6.1.2 核心函数

**cosineSimilarity(vec1, vec2)**
- 功能：计算两个向量的余弦相似度
- 公式：cosθ = (A·B) / (||A|| × ||B||)
- 返回值：0 ~ 1，值越大越相似

```typescript
function cosineSimilarity(vec1: number[], vec2: number[]): number {
  const dotProduct = vec1.reduce((sum, val, i) => sum + val * vec2[i], 0);
  const norm1 = Math.sqrt(vec1.reduce((sum, val) => sum + val * val, 0));
  const norm2 = Math.sqrt(vec2.reduce((sum, val) => sum + val * val, 0));
  return norm1 && norm2 ? dotProduct / (norm1 * norm2) : 0;
}
```

**normalizePlaytime(playtimeHours)**
- 功能：归一化游玩时长
- 规则：100 小时以上归一化为 1，小于 100 小时按比例计算

**getRecommendations(userId, games, ratings, allUserIds)**
- 功能：获取协同过滤推荐结果
- 流程：
  1. 获取用户已评分的游戏
  2. 筛选未评分且在促销中的游戏
  3. 构建用户评分向量
  4. 计算与其他用户的余弦相似度
  5. 计算每款游戏的混合评分（0.6×协同过滤 + 0.4×时长权重）
  6. 按评分排序，返回 Top N

### 6.2 AI 推荐算法

#### 6.2.1 算法架构

AI 推荐是基于内容的推荐算法，结合多维度评分：

| 维度 | 权重 | 说明 |
|------|------|------|
| 类型匹配 | 30分 | 游戏主类型是否匹配用户偏好 |
| 标签匹配 | 每标签6分 | 游戏标签与用户偏好标签匹配 |
| 内容相似度 | 最高30分 | 游戏描述文本与偏好内容相似度 |
| 促销加成 | 8分 | 促销中且折扣≥30% |
| 热门程度 | 0-20分 | Steam 好评率反映的热度 |
| 评分质量 | 0-15分 | 游戏评分高低 |
| 质量加成 | 0-8分 | 高评分游戏的额外加分 |

#### 6.2.2 核心函数

**analyzeUserPreferences(userRatings, purchasedGames, wishlistGames)**
- 功能：分析用户偏好
- 输入：用户评分、已购买游戏、愿望单游戏
- 输出：
  - preferredGenres: 偏好类型（Top 3）
  - preferredTags: 偏好标签（Top 5）
  - contentKeywords: 内容关键词集合
- 算法：
  1. 遍历已评分游戏，加权计算类型和标签偏好
  2. 游玩时长越长，权重越高（1 + playtime/100）
  3. 愿望单游戏权重为已评分游戏的 0.7 倍
  4. 从游戏描述中提取关键词，形成内容关键词集合

**extractKeywords(text)**
- 功能：从文本中提取关键词
- 处理步骤：
  1. 移除 HTML 标签
  2. 过滤停用词（中英文常见停用词）
  3. 提取英文单词（4个字母以上）
  4. 提取中文词汇（2个字以上）
  5. 去重返回

**calculateContentSimilarity(game, preferences)**
- 功能：计算游戏与用户偏好的内容相似度
- 算法：
  1. 提取游戏描述文本的关键词
  2. 与用户偏好内容关键词取交集
  3. 匹配比例 × 100 = 相似度分数
- 返回：{ score, matchedKeywords }

**calculateMatchScore(game, preferences)**
- 功能：计算游戏匹配总分
- 返回：
  - score: 综合得分（0-100）
  - matchAspects: 匹配维度说明
  - matchScore: 匹配度得分
  - popularityScore: 热门程度得分
  - ratingScore: 评分数

**排序策略**：
- 匹配度差距 > 15 分：直接按综合得分排序
- 匹配度相近（≤ 15 分）：
  - 提高评分权重（×1.5）
  - 高评分游戏可超越热度略高的游戏
  - 实现"评分显著高则优先推荐"

**getAIRecommendations(request)**
- 功能：获取 AI 推荐结果
- 流程：
  1. 分析用户偏好（已购买 + 评分 + 愿望单）
  2. 从候选池中排除已购买游戏
  3. 计算每款游戏的匹配分数
  4. 应用排序策略
  5. 返回 Top K 推荐结果
  6. 生成 AI 分析文本

#### 6.2.3 推荐去重机制

- 已推荐游戏 ID 保存到 localStorage（key: zhiyou_recommended_games）
- 每次推荐时自动排除已推荐过的游戏
- 最多保留 200 条历史记录，自动滚动淘汰

## 7. Steam API 集成详细设计

### 7.1 Steam Store API (steamSearch.ts)

#### 7.1.1 游戏搜索

**searchSteamGames(keyword, count, offset)**
- 接口：GET https://store.steampowered.com/api/storesearch
- 参数：
  - term: 搜索关键词
  - l: 语言（schinese）
  - cc: 国家（cn）
  - count: 返回数量
  - start: 偏移量
- 返回：SteamSearchResult[]
- 处理：
  1. 调用搜索 API
  2. 解析返回的 JSON 数据
  3. 转换为统一的搜索结果格式

#### 7.1.2 游戏详情

**getSteamGameDetails(appId)**
- 接口：GET https://store.steampowered.com/api/appdetails
- 参数：
  - appids: App ID
  - l: 语言
  - cc: 国家
- 缓存策略：
  - 优先从 localStorage 缓存读取
  - 缓存未命中才请求 API
  - 请求成功后写入缓存

#### 7.1.3 精选游戏

**fetchFeaturedGames()**
- 接口：GET https://store.steampowered.com/api/featuredcategories
- 返回：
  - specials: 正在促销的游戏
  - top_sellers: 畅销游戏
  - new_releases: 新发售游戏

### 7.2 图片代理

由于 Steam 图片存在跨域和 ORB 问题，使用代理服务：

**proxyImage(url)**
- 主代理：images.weserv.nl
- 备用代理：corsproxy.io
- 功能：代理 Steam 图片，解决 CORS 和 ORB 问题

**proxyImageFallback(event)**
- 功能：图片加载失败时的备用处理
- 策略：主代理失败切换到备用代理，最终失败显示占位图

### 7.3 跨域解决方案

| 方案 | 用途 |
|------|------|
| Vite Proxy | 开发环境代理 Steam API |
| corsproxy.io | 生产环境通用 CORS 代理 |
| images.weserv.nl | 图片专用代理 |
